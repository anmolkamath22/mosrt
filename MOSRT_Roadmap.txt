PROJECT OVERVIEW

Goal

Build a userspace mini operating system runtime that implements:

-   Process abstraction (PCB + state machine)
-   Preemptive CPU scheduling (RR, Priority, MLFQ)
-   Timer interrupt simulation
-   I/O blocking model
-   IPC mechanisms
-   Synchronization primitives
-   Metrics + trace logging
-   Extensible architecture for future memory/storage modules

This project maps directly to: Process Management, Scheduling, IPC,
Synchronization (Silberschatz).

Objectives

PHASE 2 — GREEN THREADS + CONTEXT SWITCHING

Use: - getcontext() - makecontext() - swapcontext()

Each process gets its own stack.

Implement: void yield();

Definition of Done: - Two processes alternate via yield() - Clean
context switching without crashes

------------------------------------------------------------------------

PHASE 3 — TIMER + CONTROLLED PREEMPTION

Use: setitimer(ITIMER_REAL) SIGALRM handler

Signal handler must: - increment global tick - set need_resched flag

Preemption occurs at safe yield points.

Accounting: - Increment cpu_time for RUNNING - Increment wait_time for
READY - Wake BLOCKED processes when wakeup_tick reached

------------------------------------------------------------------------

PHASE 4 — WORKLOAD + I/O MODEL

Workload Example:

CPU 30
IO 50
CPU 20
EXIT

CPU N = run N ticks
IO N = block until current_tick + N

------------------------------------------------------------------------

PHASE 5 — ADVANCED SCHEDULERS

Scheduler Interface: - init_scheduler() - enqueue(pid) - dequeue() -
on_tick(pid) - pick_next() - on_process_block(pid) -
on_process_exit(pid)

Implement in order: 1. FCFS 2. Round Robin 3. Priority (preemptive) 4.
Aging 5. MLFQ

MLFQ Requirements: - Multiple queues - Increasing quantum per level -
Demotion on quantum exhaustion - Periodic boost

------------------------------------------------------------------------

PHASE 6 — IPC SYSTEM

Implement Message Queue: - Bounded buffer - send(pid, msg) - recv()

Blocking semantics required.

Optional: Pipe-style channel.

PHASE 7 — SYNCHRONIZATION

Implement: - Counting semaphore - Mutex - Optional: condition variable

Required demos: - Producer–Consumer - Readers–Writers
